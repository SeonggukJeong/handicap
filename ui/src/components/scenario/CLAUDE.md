# 시나리오 에디터 (`ui/src/components/scenario/`, `ui/src/scenario/`) 함정

이 파일은 `ui/src/components/scenario/` 파일을 건드릴 때 자동 로드되는 2단계 중첩 CLAUDE.md다. 전역 규칙은 루트 `CLAUDE.md`, UI 공통(빌드 게이트·오프라인·테스트 인프라·API client)은 `ui/CLAUDE.md` 참고.

> FlowOutline(아웃라인 트리)·Monaco YAML 뷰·YAML 양방향 sync(`yamlDoc.ts`)·Zustand `useScenarioEditor` store·test-run 패널을 다룬다. **모델 로직(`ui/src/scenario/model.ts`/`cast.ts`/`scanVars.ts` 등)도 이 도메인이지만 그 디렉토리는 `components/scenario/`의 형제(자식 아님)라 그 파일만 단독으로 건드리면 이 문서가 자동 로드되지 않는다** — 에디터 슬라이스는 거의 항상 모델+컴포넌트 계층을 같이 건드리므로 실무 영향은 적지만, 순수 모델-only 편집 시엔 이 파일을 수동 참고할 것. `ui/CLAUDE.md` 분할(2026-07-07, 당시 ~143KB)에서 분리.

## 에디터 아웃라인 (`FlowOutline`, ADR-0044) · 구 React Flow 캔버스(제거됨)

> **React Flow(`@xyflow/react`)는 ADR-0044(editor-flow-outline-redesign)에서 에디터에서 완전 제거** — 현 에디터 1차 표현은 `FlowOutline`(세로 들여쓰기 HTML 트리, `ui/src/components/scenario/FlowOutline.tsx`)이고 그룹내 재정렬은 dnd-kit(`reorder.ts::resolveDragEnd`→`moveStep`). 아래 *React Flow/CanvasView/StepNode* 불릿(패키지/NodeProps/fitView/measureStep/parentId 등)은 **역사적 참조**(컴포넌트 삭제됨); 모델 헬퍼 불릿(`flattenHttpSteps`/`findStepSiblings`/`findStepById`/discriminated union)·Inspector fieldset `min-w-0`은 여전히 유효.

- **dnd-kit 그룹내 드래그 라이브 검증 — KeyboardSensor가 Playwright 자동화에서 미작동** (editor-flow-outline-redesign): `FlowOutline` 행 드래그 핸들에 focus→Space(집기)→ArrowDown→Space(놓기) 키보드 시퀀스가 Playwright-MCP에서 `onDragEnd`를 *안* 발화(순서 무변경) → **`browser_drag`(포인터 드래그, 핸들 행→타깃 행)로 검증**. 단위 테스트는 순수 `resolveDragEnd(steps,activeId,overId)`(`reorder.ts`)가 wiring을 커버(flat/null/cross-group/nested-loop/parallel-lane 6 케이스), 실제 픽셀 드래그→YAML 모달 반영만 Playwright 실측.
- **grid를 뷰포트 높이로 캡해 열별 내부 스크롤을 얻으려면 `max-h`+`min-h-0`만으론 부족 — 명시 shrinkable 행 트랙 `grid-rows-[minmax(0,1fr)]` 필수** (editor-varpanel-viewport-polish #4, F1): flex의 `min-h-0` 트릭은 flex에서만 통하고 grid의 암시적 `auto` 행은 `max-h` 캡을 무시하고 콘텐츠로 넘쳐 열의 `overflow-auto`가 발동 안 한다(넘친 열이 아래 섹션 위로 겹침) → 비-wide `editor-grid`에 `max-h-[calc(100vh-<chrome>)] grid-rows-[minmax(0,1fr)]`(기존 `min-h-[680px]` 제거)+열 컨테이너 `overflow-auto min-h-0`. **변수 aside는 `overflow-visible`**(내부 `<ul>`만 스크롤)로 224px HelpTip이 좁은 열 밖으로 미클립. **라이브 실측 함정**: 열 컨테이너의 `scrollHeight===clientHeight`여도 스크롤이 *더 깊은* 자식(FlowOutline 자체 `flex-1 overflow-auto`)에서 일어날 수 있다 — 실제 스크롤러를 descendant walk로 찾아 `scrollH>clientH` 실측(캡은 `editor-grid` `getBoundingClientRect().height ≤ innerHeight−chrome`로·`document.scrollHeight`는 아래 test-run 섹션에 오염).
- **`max-h-[calc(100vh-Nrem)]`은 상한일 뿐 — 짧은 내용에도 패널이 뷰포트를 *채우게* 하려면 `h-[calc(...)]`(fill)** (editor-viewport-polish-v2, 사용자 도그푸딩): `max-h`+`grid-rows-[minmax(0,1fr)]`(위 #4)는 내용이 캡을 넘칠 때만 캡·열 내부 스크롤을 주지만, 내용이 짧으면 그리드가 `min-h-[520px]` floor에 머물러 변수·스텝·인스펙터 패널이 짧게 뜬다("한 화면에 넓게 안 들어옴" 사용자 지적). `EditorShell` `capClass`를 `h-[calc(100vh-Nrem)]`(fixed fill)로 바꾸면 짧은 내용에도 뷰포트 높이(900px 화면→644·접힘 724)를 채우고, `min-h-[520px]`가 작은-화면 floor(calc<520이면 min-height가 height 이김), 넘치는 내용은 여전히 열 `overflow-auto`가 내부 스크롤. `capClass` 단일소스라 비-wide/varsWide 그리드+wide aside/container 4곳 동시. jsdom은 layout 0이라 클래스 계약(`h-[calc...]` 토큰)으로 락인, 실측(`getBoundingClientRect` 520→644)이 권위.
- **`className` 토큰 단언은 `.split(/\s+/)`+`toContain` — raw `toContain("h-[calc...]")`는 `max-h-[calc...]`가 substring 포함이라 false-green** (editor-viewport-polish-v2): `"max-h-[calc(100vh-16rem)]".includes("h-[calc(100vh-16rem)]")===true`라 max-h→h fill 전환 테스트에서 raw 문자열 `toContain`은 옛 max-h에도 통과해 RED가 안 뜬다(TDD 무력). `className.split(/\s+/)`로 토큰 배열 만들어 정확-토큰 membership(`toContain`/`not.toContain`)으로 단언해야 `max-h-`⊃`h-` 구분·real RED.
- **`AutoGrowTextarea`는 폭 변경에 반응해야 — `useLayoutEffect([value])`만으론 열 리사이즈 시 stale 높이** (editor-viewport-polish-v2, varsWide 노출): auto-grow가 `value` 변경 시에만 `scrollHeight`로 재계산하면 열 폭이 바뀔 때(varsWide/wideOpen 토글로 변수 열 210px↔1fr) 같은 값이 더 적은 줄로 재배치돼도 높이가 안 따라와 2줄→1줄이 돼도 2줄 높이 유지(사용자 도그푸딩 적발). `ResizeObserver`로 폭 변경 시 `resize()` 재실행 — **height-set 되먹임 루프는 `contentRect.width !== lastWidth` 너비-gate로 차단**(height 변경은 RO 재발화시키나 width 불변이라 no-op)·`disconnect` cleanup. jsdom은 `test/setup.ts`가 RO를 no-op stub으로 폴리필(콜백 미발화)이라 기존 테스트 무영향 — 신규 테스트는 `vi.stubGlobal`로 RO mock해 콜백 수동 발화.
- **글로벌 `* { scrollbar-width: thin }`은 Chromium에서 `::-webkit-scrollbar { width: Npx }`를 무력화 → 실폭 11px(스펙 8px 아님)** (editor-viewport-polish-v2 A, 라이브 실측): 최신 Chromium은 `scrollbar-width`가 non-auto면 표준 얇은 바(≈11px)를 쓰고 `::-webkit-scrollbar` 폭/thumb 규칙을 문서 전역에서 무시(A/B 프로브 실측: thin=11·auto=15, 8px 도달 불가). thumb 색은 표준 `scrollbar-color`가 담당(webkit thumb 규칙 死). 정확 8px+webkit thumb 원하면 `scrollbar-width`/`scrollbar-color` 제거하고 webkit 전용(Firefox 얇은 바 상실 트레이드오프). 얇아지긴 함(15→11).
- **드래그 검증은 *드롭 후* DOM 순서만으론 부족 — 재정렬 *중* 형제 항목의 shift-transform을 실측해야** (editor-drag-fixes): #3/#4 자동검증이 최종 순서+드래그되는 오버레이만 봐 PASS했으나 형제 컨테이너 자식의 미추종(Problem 1/2)을 못 봐 잠복→사용자 수동 적발([[implementation-rigor-over-spec]] #5 false-PASS의 드래그-프리뷰 확장). **held 드래그**(`browser_run_code_unsafe`로 `page.mouse.down→move→관측→up`) 중 형제 항목 `getBoundingClientRect` top-delta로 컨테이너 자식이 헤더와 같이 미는지(자식 `dTop`==헤더 `dTop`)·오버레이 포털에 자식 서브트리 존재·소스 wrapper `opacity-0`를 실측. **dnd-kit `setNodeRef`(측정/충돌 rect)와 적용 `transform`은 독립** — 컨테이너 sortable 노드를 헤더+밴드 외곽 wrapper로 두면(transform이 컨테이너 전체 이동=자식 추종) 충돌 rect도 키 커져 드롭이 *자식 위치*로 결정돼 비직관 → 드롭 대상 선택만 커스텀 collision(`nearestByHeader`: 컨테이너=헤더 띠 중심·leaf=행 중심을 `pointerCoordinates.y`와 비교, 그룹-스코프 후보 한정)으로 분리해 *헤더(부모) 근접도*로 결정(측정 rect ≠ 드롭 결정).
- **에디터 드래그 라이브 검증 잡 메커니즘 3종** (editor-drag-fixes): ① **vite dev는 IPv6 `[::1]`만 바인드** → Playwright `127.0.0.1`은 `ERR_CONNECTION_REFUSED`, `localhost`로 navigate(curl도 `localhost`/`[::1]`); ② **비-컴포넌트 export를 컴포넌트 파일에 두면 `react-refresh/only-export-components` warn**(`--max-warnings=0`=실패) → 순수 헬퍼는 별 모듈(`reorder.ts` 등)이 정석이나 plan 파일-스코프 유지 시 `// eslint-disable-next-line`(warn 발화=directive 사용됨=unused-directive 아님); ③ **컨테이너 child-add 버튼(`+ 반복 안에 HTTP 스텝`)은 컨테이너 *선택* 시만 디테일 패널에 노출**(top-level add bar와 별개) — Playwright 시나리오 구성 시 컨테이너 추가→자동선택 상태에서 클릭.
- **FlowOutline 드래그 시각(인디케이터·밴드 하이라이트)의 same-band 미표시는 버그가 아니라 게이트** (editor-reparent-dnd): same-band 드롭은 N1 핀으로 half를 무시하고 `computeReorder` verbatim 착지라 half-기반 인디케이터가 실제 착지와 모순될 수 있어 `overBandKey !== activeBandKey`일 때만 렌더(sortable shift가 same-band 신호·키보드 경로는 항상 same-band라 0) — 게이트를 풀면 "위로 라인 그리고 아래 착지" 모순이 돌아온다. 또 밴드 키 계약(`"top"`/`` `${parentId}:${band}` ``·placeholder `band:{parentId}:{band}`·elif 0-based)은 `dropRules.bandKey`↔`resolveDrop` 정규식↔`yamlDoc.bandSeqPath`↔`EmptyBandDrop` droppable id **4곳 동일 유지** — 한 곳만 바꾸면 드롭이 조용히 무효/오배치된다.
- **워크트리 `--ui-dir` cwd-drift → Monaco 동적 청크 MIME 에러(앱 버그처럼 보이나 서빙 아티팩트)** (editor-flow-outline-redesign 라이브검증): controller가 상대 `ui/dist`를 *프로세스 cwd*로 해석 → Bash cwd가 메인 체크아웃으로 드리프트하면 메인 dist(다른 vite 청크 해시)를 서빙, 워크트리 `index-*.js`가 lazy-import하는 `yaml-*.js`가 거기 없어 `index.html`(text/html)로 폴백 → `Failed to load module script: …MIME "text/html"` 3연발(+브라우저 디스크 캐시 잔류). **앱 코드 회귀 아님** — 절대경로 `--ui-dir "$PWD/ui/dist"`로 재시작 후 워크트리 dist 자기일관(`index-*.js`가 참조하는 `yaml-*.js`가 `200 text/javascript`) 직접 재확인으로 판별. (기존 stale-dist/포트선점 함정 군의 청크-해시 변종.)
- **Monaco 모달 라이브검증 — 프로그램으로 에디터 내용을 못 set한다(`window.monaco` 미노출 + 멀티라인 fill auto-indent 오염)** (editor-modal-layout-fixes): `@monaco-editor/react`는 `window.monaco`를 전역에 안 박아 `monaco.editor.getModels()[0].setValue(yaml)` 불가. 그렇다고 `.monaco-editor textarea.inputarea`에 멀티라인 YAML을 `fill`하면 Monaco auto-indent가 줄마다 들여쓰기를 **누적-오염**시켜 YAML이 깨진다(단일 토큰 fill은 OK·view-line 오버레이가 textarea click을 가로채니 `.view-lines` 클릭으로 focus). → 시나리오는 **실 UI로 구성**(템플릿 선택→변수 패널 `추가`·스텝 디테일 에디터 `이름`/`URL` 필드 fill, React controlled라 `fill`이 onChange 발화). **#5류 모달/에디터 *높이* 같은 시각 표면은 `getBoundingClientRect().height`(>임계) + 컨테이너 truncate는 `clientWidth<scrollWidth`로 단언 — DOM-존재/내용-텍스트만으로 PASS 금지**(직전 슬라이스가 클립보드-only로 #5 false-PASS). 에디터 `/scenarios/new`는 클라이언트-only(백엔드 불필요)·템플릿 picker 먼저 뜸.
- **Monaco 모달 내용 *읽기*도 불신 — `.view-line` DOM 추출은 가상화로 렌더 밖 줄을 놓친다** (editor-var-tools 라이브검증): rename 라이브검증에서 `[...document.querySelectorAll('.view-line')]` 조인이 실재하는 헤더 참조 줄(`Authorization: "Bearer {{auth}}"`)을 못 담아 `referenceValueLines: []`로 **거짓 부재**를 보였다(`window.monaco` 미노출이라 `getModels()[0].getValue()`도 불가·textarea 프록시 read도 부분값). YAML 모달로 재작성 결과를 검증하지 말고 — 참조/값 실측은 **스텝 폼 필드**(React controlled `input.value`, 예: `[aria-label="header value 0"]`) 또는 **파생 패널 상태**(변수 행 `auth`가 1-ref·미정의 `token` 행 *부재*로 재작성을 간접 입증)로. 둘 다 파싱된 store model 기준이라 신뢰(Monaco 표시층 ≠ 진실). 위 "못 set한다"의 read-side 짝.
- **`@xyflow/react` v12의 패키지 이름 변경** (Slice 3): 이전 `reactflow` 패키지가 `@xyflow/react`로 rename. import 경로도 `@xyflow/react` + `@xyflow/react/dist/style.css`. v11 예제는 함정.
- **`@xyflow/react` v12의 `NodeProps` 시그니처** (Slice 3): v11의 `NodeProps<Data>` 가 아니라 `NodeProps<Node<Data, "type-string">>` 형태. `node_modules/@xyflow/react/dist/esm/types/index.d.ts` 확인 후 맞춰야 함.
- **React Flow의 control vs uncontrolled** (Slice 3): 노드 위치를 직접 계산해서 넘기면 React Flow 안에서 drag로 옮긴 위치는 반영되지 않는다. Slice 3은 의도적으로 drag 비활성화(`draggable: false`) — 위치는 매번 재계산됨.
- **`<ReactFlow fitView>` prop은 init 1회만 fit한다 — 비동기 로드 노드엔 무력** (canvas-fix): `EditorShell.loadFromString`이 post-mount effect라 첫 페인트엔 `steps=[]` → fitView가 빈 그래프를 fit하고, 진짜 노드가 도착해도 재fit 안 함 → 기존 시나리오 진입 시 노드가 화면 밖 lower-left에 박혀 "엉뚱한 곳" 표시(로드 타이밍에 따라 간헐적=race). 해결: `<ReactFlow>` 자식으로 `AutoFitView`(=`useNodesInitialized()` + `useReactFlow().fitView()`)를 두고, **노드 id join을 `fitKey`로 삼아** 구조 변경 시에만 1회 재fit(수동 pan/zoom 보존). `useNodesInitialized`는 측정 완료 전 false라 측정 후에만 fit. 큰 그래프가 default `minZoom`(0.5)에 막혀 다 안 들어오면 `minZoom={0.2}`. jsdom 테스트는 노드 측정이 없어 `useNodesInitialized`=false → fit 미호출(무해, 기존 CanvasView 테스트 그대로 통과).
- **React Flow v12 parent/child(subflow)** (Slice 7): 자식 노드에 `parentId` + `extent: "parent"` 를 주고, **부모 노드에 명시적 `style` width/height** 를 줘야 자식이 컨테이너 bounds 안에 담긴다(부모 크기는 자동 산출 안 됨 — 자식 수에 맞춰 높이 계산). full `<ReactFlow>` 를 jsdom 에서 마운트하는 RTL 테스트는 `ResizeObserver` 폴리필 필요(xyflow 의 ZoomPane 이 요구) — `ui/src/test/setup.ts` 에 conditional 폴리필.
- **`Step` 을 discriminated union 으로 바꾸면 모든 consumer 가 union narrowing 을 거쳐야 한다** (Slice 7): `.request`/`.assert`/`.extract` 를 직접 읽던 TS 코드가 전부 `tsc` union 에러를 낸다. `flattenHttpSteps(steps)` 가 "트리에서 http leaf 만 평탄화" 하는 표준 헬퍼(report 라벨링·inspector 중첩 선택에 재사용) — 새 컨테이너 노드(Slice 8/9) 추가 시 이 헬퍼의 walk 만 확장하면 된다.
- **`StepModel`은 이제 http|loop|if 3-way discriminatedUnion** (Slice 9b): if 컨테이너 노드는 `IfStepNode.tsx`(헤더 + 조건 요약 + THEN/ELIF/ELSE 밴드 라벨), CanvasView `summarizeCondition` + if-layout + `NODE_TYPES.if`. **`flattenHttpSteps`는 if case에 한해 비재귀** — then/elif[].then/else 한 레벨만 평탄화하고 9c(상호 1레벨 중첩, spec SF-1)까지 그대로 둔다(분기는 9b에서 http-only). 반대로 `findStepPath`(`yamlDoc.ts`)는 9c 포석으로 이미 완전 재귀.
- **`flattenHttpSteps`·`findStepSiblings`는 9c에서 완전 재귀로 전환** (Slice 9c): 두 컨테이너 타입(loop `do` + if 모든 분기) 하강. **`findStepById`** 신규 추가 — 어떤 타입의 스텝도 찾는 범용 resolver. inspector는 9b까지 `flattenHttpSteps`(http leaf만 반환)로 step을 resolve했기 때문에 중첩 컨테이너(NestedLoop/NestedIf)가 클릭해도 inspector에 안 잡혔다 — 9c에서 `findStepById`로 교체해 해소.
- **CanvasView 재귀 `measureStep`/`emitStep` 서브플로우 에미터** (Slice 9c): 높이는 bottom-up(내부 컨테이너의 높이가 외부 컨테이너 높이를 결정). 자식 width 는 depth 마다 `LOOP_PAD*2` 씩 축소(내부 URL이 컨테이너 밖으로 안 자라게). 중첩 자식은 React Flow `parentId`+`extent:"parent"` 지정. 죽은 `CHILD_WIDTH` 상수 제거됨.
- **parallel만 가로로 성장 — `measureWidth` 신규 + 세로 레인 에미터** (A2): loop/if는 세로 성장이라 폭 고정(`NODE_WIDTH`)이지만 parallel은 분기를 **가로로 나란히**(세로 레인) 깔아 폭이 분기 수에 비례 → `measureWidth(step)`(parallel만 `branches.length*(LANE_WIDTH+LANE_GAP)-LANE_GAP+LOOP_PAD*2`, 그 외 `NODE_WIDTH`) 신규. 최상위 루프의 x-전진을 `measureWidth`로(고정폭 아님 — 안 하면 parallel 컨테이너가 다음 노드와 겹침). `emitStep` parallel arm은 컨테이너 1개 push 후 분기 좌→우(`lx += LANE_WIDTH+LANE_GAP`), 각 분기 자식 위→아래(`ly += measureStep(child)+CHILD_GAP`)로 `parentId`로 emit. `ParallelStepNode`(헤더+레인 라벨만, 자식은 parentId로 부유)는 `interface ParallelStepNodeData extends Record<string,unknown>`(v12 제약, `IfStepNode` 패턴). **레인 라벨 y는 하드코딩 말고 `lanes:{name,x,y}[]` data로 흘릴 것**(`IfStepNode.bands.y` 패턴 — `PARALLEL_HEADER_H` 상수가 CanvasView에 있어 컴포넌트가 직접 못 읽음, 하드코딩하면 헤더 높이 바뀔 때 라벨이 어긋남).
- **여러 flex 컨텍스트에 꽂히는 공유 셀의 `<button>`은 text-align 명시 필수 — flex-col(stretch) 부모에서만 UA 기본 `text-align:center`가 드러난다** (extract-var-name-visibility, 사용자 도그푸딩 발견): `VariablesPanel.usageCell` 버튼이 flex-row 행(content-폭)에선 좌측처럼 보였지만 declared 행 `li flex flex-col`에선 full-width stretch되어 라벨이 가운데 렌더(같은 자리 0-ref `<span>`("미사용")은 좌측 = 행 내부 불일치·의도된 디자인 아님) → 공유 className에 `text-left` 1토큰(콘텐츠-폭 인스턴스 무영향·클릭 타깃 full-width 유지). jsdom은 layout-free라 RTL 클래스 계약으로 락인만 하고, 정렬 통일은 라이브 `getComputedStyle(btn).textAlign`+텍스트 좌단 rect(`Range.getBoundingClientRect`)==이름 좌단으로 실측.
- **아이콘-only 버튼의 히트영역 확대(`h-6 w-6`)는 좁은 flex-wrap 행의 wrap 폴백을 조기 트리거한다 — negative margin으로 시각 간격·행 폭을 회수** (varpanel-toggle-size 2026-07-24): 변수 패널 declared 헤더 행(토글+이름+연필+배지+×, 변수 aside ~184px)에서 토글을 24px로 키우자 gen 배지 있는 행의 ×가 줄바꿈(배지+× 묶음 wrap[badge-x-wrap-fix]은 의도된 극단 폴백이나 일상 케이스 발동은 회귀) + 히트영역 내부 여백(~8px)이 `gap-x-2`와 겹으로 쌓여 캐럿→이름 시각 간격 ~16px(사용자 "마진 너무 큼" 지적). fix = 토글 `-mr-1.5` + 밀집 행만 `gap-x-1.5`(형제 행들은 `gap-x-2` 유지). jsdom은 wrap을 못 보므로 클래스 계약 테스트로 락인하고, same-line·여유는 Playwright rect 실측(`×.top==이름.top` + 이름 폭−`min-w-[72px]`=slack — flex-1 이름이 여유를 흡수하므로 우단 정렬로는 slack이 안 보인다).
- **flexbox `min-width:auto` 오버플로우 + `truncate`는 bounded width 필요** (Slice 7-1): `flex` row에서 `flex-1` 입력 옆 버튼이 칸 밖으로 밀려나는 건 flex item 기본 `min-width:auto` 때문 — 입력에 `min-w-0`, 트레일링 버튼에 `shrink-0`. Tailwind `truncate`는 **조상에 확정 너비가 있어야** 클립된다 — React Flow 노드는 콘텐츠로 자라므로 `CanvasView`에서 노드 `style.width`를 박고 노드 root에 `w-full box-border`를 줘야 긴 URL이 컨테이너 밖으로 안 자란다. 새 key-value 폼·새 캔버스 노드 추가 시 둘 다 확인.
- **`min-width:auto` 함정은 입력뿐 아니라 *컨테이너(fieldset)* 자신에도 적용된다** (canvas-fix): Inspector(`aside flex flex-col`)의 좁은 ~294px 폭에서 `Request` `<fieldset>`이 column의 flex item이라 자신의 `min-width:auto`=min-content(332px)로 **줄어들길 거부**하고 패널 밖으로 삐져나간다(다크모드에선 패널 테두리가 보여 "파란 박스가 오른쪽으로 새는" 글리치로 보임). 내부 `flex-1` 입력에 `min-w-0`을 줘도 **fieldset 자체에 `min-w-0`이 없으면** 안 고쳐진다 — 둘 다 필요(fieldset가 column 폭으로 줄고, 내부 콘텐츠가 줄어든 fieldset에 맞음). Inspector의 모든 fieldset(Request/Assertions/Extracts/Condition/Elif)에 `min-w-0` 적용. 진단: `el.scrollWidth > el.clientWidth`로 overflow 확인 + `el.style.minWidth='0'`로 범인이 그 flex-item인지 이분 검증(Playwright `browser_evaluate`).

(Inspector think min/max — plan 의 `commitThinkTime` 의사코드엔 이 가드가 없어 implementer 가 정정, 특성화 테스트로 락인. RunDialog 의 동형 검증은 둘 다 채워야 `thinkInvalid`/submit 통과라 자연 회피.)
- **짝 입력의 draft 상태기계·커밋 시점은 `useThinkTimePair` 훅 하나가 소유한다 — 새 소비처를 손으로 복제하지 말 것** (pair-input-blur-commit 2026-07-20, `300e294`): `useThinkTimePair({value, resetKey?, onCommit, onClear}) => {minProps, maxProps, reseed}`. `minProps`/`maxProps`를 `<Input>`에 **스프레드**하면 끝(`value`/`onChange`/`onBlur`/`ref` 4개; 나머지 props는 그대로 둔다). 타입 함정: `ThinkPairFieldProps.ref`는 `RefObject<HTMLInputElement>`여야 한다 — `RefObject<HTMLInputElement | null>`로 쓰면 `readonly current` 공변 때문에 네이티브 `<input>` 스프레드에서 **`tsc -b`만** TS2322로 터진다(`pnpm lint`/`pnpm test`는 통과 — `pnpm build`까지 돌려야 보인다). `resetKey`는 **편집 대상이 바뀌는 표면만** 넘긴다(`step.id`/`row.stepId`); 시나리오 하나뿐인 기본값 표면은 생략(identity가 없다). 재시드 dep은 **원시값**(`value?.min_ms`/`value?.max_ms`) — 객체 dep이면 표에서 한 행 커밋이 다른 행의 반쯤 친 draft를 날린다. `reseed`는 `useCallback` 안정성이 필수(소비처가 effect dep으로 쓰고 `pnpm lint`는 `--max-warnings=0`). 모달 안 소비처는 **닫기 flush도 함께** 배선한다 — ESC는 포커스 이벤트 없이 언마운트해 blur가 안 뜨므로, 보류 규칙만 넣으면 ESC에서 두 칸 모두 유실된다(`closeWithFlush` = 동기 `activeElement.blur()` 후 `onClose`, `EditorShell.closeDetail`과 같은 이디엄). 백드롭·✕는 mousedown이 이미 포커스를 옮겨 스스로 커밋하므로 **ESC만이 이빨 있는 경로**다.
- **조건 leaf의 `right`는 `exists`/`empty`에서 반드시 생략** (Slice 9b): 엔진이 이 두 단항 연산자에서 `right`를 떨군다 — `yamlDoc.ts::cleanCond`가 `createNode` 전에 `right`를 OMIT(다른 연산자는 편집 가능하게 `right:""`도 유지)하고, leaf inspector도 op-change 시 `right`를 떨군다. 두 군데 다 안 지키면 round-trip에 죽은 `right` 키가 남는다.
- **`findStepPath`는 이제 완전 재귀(loop `do` + if 분기 then/else/elif[].then)** (Slice 9b): 9c(중첩) 포석. 단 step-tree edit 6종(addIfStep/setIfCond/setElifCond/addStepInBranch/addElif/removeElif)은 `BranchSel`/`branchPath`로 분기를 가리키고, `normalizeStep`(if)+`normalizeElif`가 if/elif 노드를 정규화. store action은 loop action을 미러한 thin wrapper.
- **`addIfInLoop`/`addLoopInBranch` Edit 변형 + store action** (Slice 9c): 중첩 컨테이너를 seeded http 자식 1개와 함께 삽입한다. http 자식을 이미 있는 컨테이너에 추가하는 `addStepInBranch`/`addStepInLoop`(9b)는 재귀 `findStepPath`가 중첩 경로를 찾아주므로 별도 edit 변형 불필요 — 신규 컨테이너 생성 2종만 추가됨.
- **변수 rename 재작성 정규식은 `splitFlowToken` base 판정과 *대칭*이어야 — lookahead가 넓으면 비-키워드 `:suffix`/공백 이름을 오염** (editor-var-tools): `renameVariable` (c) 패스가 `{{old}}`의 base만 `{{new}}`로 바꾸는데, 최초 lookahead `(?=[:}\s])`는 `old` 뒤 *아무* 콜론/공백이나 토큰 경계로 봐 `{{old:foo}}`(foo 비-키워드)·`{{old bar}}`(공백 이름)까지 재작성했다. 그러나 scan측 `splitFlowToken`은 trailing `:kw`(kw∈`CAST_KEYWORDS`)만 cast로 떼므로 이들의 base는 `old:foo`/`old bar`(≠`old`) — **scan↔rewrite 비대칭**(spec R6 "base 정확일치" 위반, 드문 콜론/공백-이름 변수에서만 발현이라 저-도달이지만 시나리오 조용히 손상). 좁힌 lookahead `` `\{\{(\s*)${escapeRegExp(old)}(?=\s*\}\}|\s*:\s*(?:${CAST_KEYWORDS.join('|')})\s*\}\})` ``로 base가 정확히 oldName일 때만 매치(cast 있으면 유효 키워드+`}}`, 없으면 `}}`)·`CAST_KEYWORDS`를 `cast.ts`에서 import해 by-construction 대칭·드리프트-free. 기존 (a)-(d) `yamlDoc.rename` 테스트·RTL은 이 비대칭을 못 잡음(유효 케이스만 검증) — **whole-branch `handicap-reviewer`(Opus)가 적발**, orchestrator node 배터리 11/11(mixed-scalar `/a/{{old}}/b?x={{old:foo}}&y={{old:num}}` 포함)로 독립 실증.

- **`doc.setIn(["새_최상위_키"], …)`은 루트 맵 *끝*에 append한다 — 라이브 검증에서 YAML 앞 N줄만 읽고 "안 쓰였다"로 오진 금지** (think-time-defaults): Rust `Scenario` 구조체의 필드 순서(`cookie_jar`↔`steps` 사이)와 달리 GUI가 쓴 새 루트 키는 **문서 맨 아래**(steps 뒤)에 나타난다. Playwright에서 `.view-line`을 `slice(0,7)`만 떠서 보고 "write-only 버그"로 오판할 뻔했다(실제로는 정상 기록). Monaco 모달 read는 **전체 `.view-line`**을 뜰 것(+ 가상화로 렌더 밖 줄을 놓치는 기존 함정과 별개 — 둘 다 "모달 read 불신"의 사례). 기존 키 덮어쓰기는 위치를 유지하므로 first-write만 해당.
- **에디터 왼쪽 패널/인스펙터에 *새 편집 어포던스*를 추가하면 `yamlError !== null` disabled 게이트가 필수** (think-time-defaults S1): store의 `dispatch`가 깨진 YAML 버퍼에서 early-return no-op이라(`store.ts`), 입력만 활성으로 두면 사용자가 친 값이 **조용히 삼켜지고** 이어서 재시드 `useEffect`가 draft를 되돌려 "입력이 사라진" 것처럼 보인다. `VariablesPanel`/`FlowOutline`은 이미 그 상태에서 편집 어포던스를 전부 잠그고 빨간 `ValidationBanner`를 띄운다 — 새 컴포넌트도 같은 이디엄(`const yamlError = useScenarioEditor((s) => s.yamlError)` + `disabled={yamlError !== null}`)을 따를 것(읽기 전용 크롬[접기 토글]은 활성 유지). RTL 락인: 깨진 YAML을 `setPendingYamlText`+`commitPendingYaml`로 만든 뒤 `toBeDisabled()`(그 경로는 `doc`/`model`을 **보존**하므로 입력이 DOM에 남아 단언이 성립).

## Zustand store

- **Zustand v5는 getInitialState 미제공** (Slice 3): 테스트에서 store를 reset하려면 직접 INITIAL 객체를 보관하고 setState로 덮어쓰는 헬퍼가 필요. 액션 ref는 v5에서 stable하므로 모듈 로드 시 한 번만 `getState()`로 캡쳐.
- **`removeStep`은 selection clear가 dispatch보다 먼저** (Slice 3): 순서를 반대로 하면 subscriber가 잠깐 "삭제된 step을 가리키는 selectedStepId" 상태를 본다 → Inspector가 stale step을 deref. store action에서 `if (get().selectedStepId === stepId) set({ selectedStepId: null })`를 dispatch보다 먼저 호출.
- **`@testing-library/react` + Zustand의 store reset 패턴** (Slice 4): 각 `it` 전에 `useScenarioEditor.setState(useScenarioEditor.getInitialState())`로 초기화. RTL는 React 트리만 재마운트하므로 모듈 스코프 store는 직접 비워야 한다.
- **셀렉터의 인라인 `?? {}`/`?? []` fallback 금지 — 모듈 스코프 안정 상수로** (getSnapshot fix 2026-06-12): `useScenarioEditor((s) => s.model?.steps ?? [])`는 model=null 동안(EditorShell pre-load 윈도) 매 스냅샷 새 객체 → React "getSnapshot should be cached" 경고, **단독 마운트면 Maximum update depth 크래시**(VariablesPanel 핀 테스트가 재현). `EMPTY_STEPS`/`EMPTY_VARS` 모듈 상수 사용. **이 경고는 react-dom 모듈 수명당 1회(warn-once)** — 콘솔 스파이 핀 테스트는 그 파일의 *첫* 마운트여야 결정적(EditorShell.test.tsx 첫 it 주석 참고). 렌더 본문의 일반 파생(`data?.x ?? []`)은 무관 — 셀렉터(getSnapshot) 안만 해당.
- **다른 패널에서 일어나는 이산 이벤트로 열린 스텝의 draft를 재시드하려면 content-deep-compare가 아니라 *성공에만 증가하는 스칼라 epoch*를 재시드 dep에** (editor-varpanel-viewport-polish #5): 변수 rename이 열린 스텝의 헤더/JSON바디/추출 draft를 실시간 갱신해야 하는데, draft를 미러 콘텐츠 deep-compare로 재시드하면 필드 *자체* 커밋마다 재발화해 진행 중 편집을 리셋한다(KeyValueGrid resetKey 함정). 해결: store에 `renameEpoch: number`(성공 rename `set`에서만 `+1`·no-op/실패 분기 미증가·시그니처/반환타입 불변·additive 클라 상태·와이어 무접촉)를 더해 `JsonBodyField`/`ExtractEditor` reseed `useEffect` dep + `KeyValueGrid` `resetKey={`${step.id}:${renameEpoch}`}`에 배선. self-commit엔 epoch 불변→재시드 0(rename은 변수 패널 클릭이라 그 순간 Inspector 필드는 blur됨). **재시드 셀렉터는 재시드하는 4 하위컴포넌트 *각각* 안에** — 최상위 Inspector 본체는 그 draft를 안 들어 top-level 셀렉터면 unused(lint)+`resetKey` out-of-scope(tsc). URL/메서드는 store-controlled(draft 없음)라 이미 live·조건편집기는 `[cond]` content 재시드라 이미 live.

- **id-반환 store 액션의 반환타입을 넓히면(`string`→`string|null`) id를 *직접 소비*하던 기존 테스트 call-site가 `tsc -b`에서 깨진다** (editor-gate-errpct-fixes): 편집 게이트가 add 액션 9종을 locked(`yamlError!==null`) 시 `null` 반환으로 바꾸며 반환타입을 `string|null`로 확장하자, `const id = getState().addLoopStep(...)`처럼 id를 바로 쓰던 `store.test.ts`·`Inspector.test.tsx` 13곳이 `TS2345`(null 비할당)로 깨졌다. `!` non-null 단언(`addLoopStep(...)!`)으로 fix — 그 테스트들은 항상 `yamlError=null` valid state에서 도니 non-null이 보장돼 안전. `pnpm test`(esbuild)는 통과·**`pnpm build`(`tsc -b`)만 적발** → src 0-diff 슬라이스여도 이 test-파일 수정이 UI 게이트에 걸리므로 같은 커밋에 fold(브리프의 "2 파일만 add" 지시를 초과해도 green 빌드 유지가 우선). 편집 게이트 자체는 `dispatch`(applyEdit 前)·`reparentStep`(clone 前)·`removeStep`(selection-clear 前) 진입 가드가 거의 모든 구조적 편집을 no-op으로 덮고, id-반환 add만 별도로 `null`을 돌려 `select(null)`(이미 `string|null` 허용)로 phantom-select을 막는다(형제 `addStepExtract` 가드의 일반화).

## 시나리오 에디터 test-run 패널 (C-2, `TestRunPanel`)

- **응답기반 extract 작성은 `onAddExtract` prop 경유** (응답기반 extract 작성): `TestRunPanel`은 프레젠테이셔널 유지 — 부모 `TestRunSection`이 `onAddExtract={(id,ex)=>useScenarioEditor.getState().addStepExtract(id,ex)}`로 store 연결. 본문 트리(`ResponseBodyTree`)는 **응답 본문이 valid JSON && !truncated일 때만**(짧으면 BodyBlock 인라인, 길면 BodyViewer 모달 트리 토글), 스칼라 leaf만 +추출. 헤더/쿠키/상태는 응답 측 HeaderTable/버튼만(요청 측 affordance 없음). 생성 JSONPath는 `jsonPath.ts`(RFC 9535, 제어문자 `\uXXXX` — serde_json_path가 raw 제어문자 거부). 확인행(`ExtractConfirmRow`)은 평범한 인라인 JSX(중첩 Modal/HelpTip 금지 — ESC 함정).
- **`ScenarioTraceSchema`는 엔진 `crates/engine/src/trace.rs`와 와이어 1:1** (C-2): serde가 모든 필드를 항상 emit(`Option`→`null`, 빈 map→`{}`, 빈 vec→`[]`, `skip_serializing_if` 없음)하므로 Zod는 `Option`=`.nullable()`(**`.optional()` 아님**), `extracted`/`unbound_vars`/`headers`/`set_cookies` 같은 항상-emit 컬렉션은 **required(`.default()` 금지** — `.default()`면 부모 `z.infer`에 `T|undefined` 누출, `pnpm build`에서만 잡힘). `StepKind`는 lowercase `"http"`/`"if"`(엔진 `rename_all="lowercase"`). 엔진 `trace.rs`에 필드/`StepKind` 추가하면 이 스키마 + `TestRunPanel` 렌더 둘 다 갱신.

- **Test-run 컨트롤+패널은 `TestRunSection`으로 추출돼 edit·new 두 페이지 공유** (C-2 후속): `<TestRunSection yamlText={...}>`가 picker 상태(`selectedEnvId`/`envEntries`/`maxRequests`)·`useTestRun`·`traceSteps` 메모·`<EnvironmentPicker>`+`<TestRunPanel>`을 다 품는 자족 유닛 — 유일 입력이 버퍼라 `ScenarioEditPage`(저장된 시나리오)와 `ScenarioNewPage`(`STARTER_YAML` 미저장 초안) 둘 다 한 줄로 마운트. test-run이 stateless·ephemeral(inline YAML)이라 **미저장 초안에도 동작**(`/scenarios/new`). 처음(C-2)엔 `ScenarioEditPage`에 인라인이었고 두 번째 소비자가 생겨 추출했다. `aria-label`(`Test run controls`/`Test run result`) 보존이라 기존 RTL 셀렉터 그대로.
- **`ScenarioEditPage` 풀페이지 RTL 마운트는 Monaco 모킹 불필요** (C-2): 에디터 store 기본 탭이 `"canvas"`라 `EditorShell`이 `CanvasView`(React Flow)를 렌더(Monaco는 YAML 뷰에서만 import). 기존 `ResizeObserver` 폴리필(`test/setup.ts`)로 jsdom 마운트 가능. store가 module-scoped라 page 테스트에 둘째 `it`을 더하면 `beforeEach` store reset 필요(단일 케이스면 self-contained).
- **조건부로 `return null`하는 컴포넌트에서 측정 effect의 deps에 "그 조건"을 안 넣으면 effect가 영영 재실행 안 된다 — 게이트는 green인데 기능이 죽는다** (editor-wide-view-overflow 2026-07-18, 최종 리뷰가 적발한 블로커): `TestFlowChips`가 `steps.length===0`이면 `return null`(wrap div 없음) → 측정 `useLayoutEffect`가 `if (!el) return`으로 빠지는데 deps가 `[expandable]`(불변)이라 **다시 안 돈다** → 나중에 steps가 도착해 칩이 떠도 `ResizeObserver`가 끝내 등록되지 않아 overflow 토글이 영구 부재. **터지는 경로가 주 화면이었다**: `TestRunSection`은 `ScenarioEditPage.tsx`에서 `{seeded && …}` 게이트 **밖**에 렌더돼 `yamlText=""`로 먼저 마운트되므로 `/scenarios/{id}`는 항상 이 경로(반면 `/scenarios/new`는 템플릿 갤러리 게이트가 마운트를 시드 뒤로 미뤄 우연히 정상 — **plan의 라이브 스크립트가 `/scenarios/new`만 썼다면 false-PASS**). 수정 = `hasSteps`를 **게이트이자 dep**으로(`if (!expandable || !hasSteps) return;` deps `[expandable, hasSteps, expanded]`) — 겸사 steps→0→N 왕복 시 detached 노드를 계속 관측하던 문제도 cleanup이 돌며 해소. 교훈: 조건부 null 렌더 + ref 측정 effect는 **"ref가 붙는 조건"을 반드시 dep에**, 그리고 **라이브 검증은 컴포넌트가 마운트되는 *모든* 진입 화면에서**(한 화면만 보면 그 화면이 우연히 정상인 버그를 못 본다).
- **캡을 클래스로 걸고 RO로 재측정하는 "펼치기" 토글은 펼친 상태에서 측정을 멈춰야 — 안 그러면 접는 순간 버튼이 사라지며 키보드 포커스가 `<body>`로 유실** (같은 슬라이스, F2): 펼치면 `max-h-*`가 빠져 `scrollHeight===clientHeight` → RO가 `overflowing=false`로 덮어씀 → "접기" 클릭이 커밋되는 렌더에서 게이트 `(overflowing || expanded)`가 둘 다 false가 돼 **버튼 언마운트**(RO 틱이 곧 되살리지만 포커스는 이미 소실). 해법은 `measure`에 `if (expanded) return` 클로저 가드 + **`expanded`를 dep**으로 — `expandedRef`(ref) 변형은 금물: 재측정이 RO 틱에만 의존해, 펼친 채 내용이 캡 아래로 줄면 접을 때 박스 높이가 안 변해 틱이 안 오고 stale `overflowing=true`가 죽은 토글을 남긴다(ref는 exhaustive-deps 면제라 경고로도 안 잡힘). **jsdom으로 잡을 수 있다**: wrap의 `clientHeight`를 live `className`을 읽는 **인스턴스 getter**로 만들고 RO 콜백을 수동 발화하면 언마운트·포커스 유실이 결정적으로 재현(prototype 상수 mock으론 안 잡힘).
- **소비처가 둘인 공유 컴포넌트의 `data-testid`는 페이지 전역 유일이 아니다** (같은 슬라이스): `chip-strip-wrap`이 EditorShell wide 섹션과 `TestRunSection` 양쪽에 존재 → 라이브/RTL 쿼리는 반드시 섹션 스코프(`section[aria-label="스텝 흐름 (넓게 보기)"]` 안에서, 또는 `within(getByRole("region", …))`). 페이지-wide `getByTestId`는 조용히 엉뚱한 쪽을 집는다.
