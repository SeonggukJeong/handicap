# 리포트 렌더링 (`ui/src/components/report/`) 함정

이 파일은 `ui/src/components/report/` 파일을 건드릴 때 자동 로드되는 2단계 중첩 CLAUDE.md다. 전역 규칙은 루트 `CLAUDE.md`, UI 공통은 `ui/CLAUDE.md` 참고.

> 리포트 차트(Recharts)·다운로드(blob/File System Access API)·bySecond 시계열 도출을 다룬다. 다운로드 유틸(`ui/src/api/download*.ts`)은 형제 디렉토리라 이 문서가 자동 로드 안 되지만, 대부분 리포트/비교 화면에서만 쓰여 실무상 이 도메인과 항상 같이 다뤄진다. `ui/CLAUDE.md` 분할(2026-07-07)에서 분리.

## 리포트 렌더링 · 다운로드 · 차트(jsdom)

- **막대(waterfall) 뷰의 고정폭 라벨 div는 `truncate`+`title` 필수 — 칩(표) 뷰는 `break-all`** (스텝 막대 라벨 overflow): `StepPhaseBreakdown.tsx` 막대 뷰 각 행은 flex `[라벨 w-40][막대 flex-1][ms w-16]`인데 라벨 div가 **고정 `w-40` flex item**이라, 긴 URL(HAR 임포트 `METHOD /path`처럼 공백·하이픈 없는 한 덩어리 토큰)이 overflow 처리 없으면 막대 그래프 위로 삐져나와 겹친다 → `shrink-0 truncate`(말줄임·nowrap·overflow-hidden·`w-40` 확정폭이라 즉시 클립[truncate 조상 확정폭 규칙 충족]·`shrink-0`=flex `min-width:auto` 역방향 방어)+`title`(잘린 전체 호버 툴팁·표시 소스 `m?.name ?? s.step_id`와 동일=ko.ts 무관·사용자 데이터). 대조로 칩(표) `StepStatsTable`은 같은 `m.name`을 `<td>`(셀 자동폭+URL칸 `break-all`)로 렌더해 *구조적* 무겹침(고정폭 flex item과 대조)이라 손댈 필요 없음. **jsdom은 픽셀 overflow 미관측** → contract 테스트는 라벨 요소(`getByText(longName)`, bar `role=img` 아님)의 `truncate` 클래스+`title` 단언으로 락인, 실제 겹침 제거는 Playwright `browser_evaluate`로 `scrollWidth>clientWidth`(clip 활성)·`label.right < bar.left`(무겹침) 실측.
- **`StepPhaseBreakdown` 막대(waterfall) 뷰는 `anyPhase`(`s.wait != null || s.download != null`)일 때만 렌더 — `measure_phases:true` run이 아니면 `StepStatsTable`(표)로 fallback** (스텝 막대 라벨 overflow 라이브검증): 막대 뷰를 라이브로 보려면 run을 `measure_phases:true`로 생성해야 한다(+responder 지연 ≥50ms로 wait/download p50>0 — localhost sub-ms면 phase 0). 안 하면 phase 데이터가 없어 표가 떠 막대 뷰 자체를 검증 못 한다.
- **분기 breakdown은 별도 `BranchStatsTable`("Branch decisions" 섹션)** (Slice 9d): `if` 노드 id는 `report.steps`(http-leaf 메트릭)에 없으므로 `StepStatsTable` 와 분리된 독립 컴포넌트로 렌더. `ifMeta`는 `findStepById`(9c 추가)로 if 노드 표시명 resolve. SQL이 `branch` TEXT 사전순으로 반환하므로 UI `branchRank`로 `then < elif_n < else < none` 재정렬(SQL ORDER BY 만 믿으면 안 됨). `none` 레이블은 `(미매치)`로 표시. `ReportSchema.if_breakdown`은 `.optional()`이라 consumer는 `?? []`로 기본값 처리.
- **bySecond 시계열 derivation은 ReportView 안에서** (Slice 5): 시계열 max-over-steps 합산 같은 derivation 로직을 backend가 아니라 ReportView 안에 두기로. backend는 raw windows 만 보낸다. 이유: UI가 step 필터/색상 분리 같은 변형을 더하기 쉬움. (step 라벨링도 UI 책임 — `crates/controller/CLAUDE.md` 참고.)
- **Recharts ResponsiveContainer + jsdom** (Slice 5): ResponsiveContainer는 부모의 measured 사이즈를 읽는데 jsdom은 layout이 없어 size=0 → SVG 미생성 → RTL assertion 실패. 컴포넌트에 explicit `width`/`height` prop을 받게 하고 ResponsiveContainer는 프로덕션 path에서만. 테스트는 explicit size로.
- **Recharts `<Line type>`는 차트 종류로 갈린다 — 관측(observed) 시계열은 `monotone` OK, 설정/분위(config/percentile) 곡선은 `type="linear"` 필수** (compare-view-depth): "다단계 ramp UI" 섹션의 `StageCurvePreview` 노트("`monotone` 금지")는 **piecewise-linear 설정 곡선**(스테이지 램프)·`PercentileCurveChart`(분위 곡선)에만 적용 — monotone cubic smoothing이 직선 구간 모양을 왜곡하기 때문. 반대로 *관측된* 초당 throughput/latency 시계열(단일-run `TimeSeriesChart`·비교 오버레이 `CompareTimeSeriesChart`)은 measured 데이터라 monotone/linear 둘 다 시각적으로 정당하고, **단일-run 차트가 이미 `monotone`을 쓰므로 오버레이도 monotone으로 미러**(spec이 §4.3 자구로 `type="linear"`를 적었어도 R2의 "TimeSeriesChart 패턴 미러"가 우선 — 사용자가 보는 두 차트의 일관성). 새 Recharts 라인 추가 시 "설정/분위 곡선이냐 관측 시계열이냐"로 type을 고를 것.
- **Recharts `<Tooltip>`는 hover로만 렌더 — RTL(jsdom, hover 없음)·headless Playwright 스크린샷 둘 다 못 잡는다** (레이턴시 분포): tooltip `formatter`/`labelFormatter` 로직(예: `formatLatency(value*1000)`, 버킷 `[lower–upper]` 범위)은 `tsc -b`+코드리뷰로만 검증, 렌더 테스트는 `region`+`svg` 존재만 단언. 단 **축 틱 텍스트**(분위 `p99`·`formatLatency` 라벨)는 DOM에 렌더돼 `toHaveTextContent`로 잡힌다.
- **Recharts `<Legend>`도 `<li>`를 렌더 — 커스텀 범례를 `querySelectorAll('li')[index]`로 단언하면 차트 범례 항목을 집는다** (곡선 fan-out 워커 표시 라이브검증): `ActiveVuChart` 워커별 뷰는 recharts `<Legend>`(라인 이름 "워커 N 목표/실제")와 커스텀 `<ul>` 범례("■워커 N", `title=worker_id`)가 **둘 다 `<li>`**라, 인덱스 기반 추출이 엉뚱한 항목을 잡는다(이번엔 `lis[0].title`이 recharts 항목=null). Playwright/RTL은 **구분 속성(`title`)·텍스트로 필터**, 인덱스 금지.
- **inline `style={{backgroundColor: hex}}` 색 단언은 `toHaveStyle({ backgroundColor: hex })`로 — `getAttribute("style").includes(hex)`(hex-substring)는 항상 실패** (compare-view-polish): 이 repo jsdom은 inline `style={{backgroundColor:"#2563eb"}}`을 `getAttribute("style")`에서 `"background-color: rgb(37, 99, 235);"`로 직렬화(hex 문자열 미포함)한다 → hex-substring 검사는 통과/실패가 뒤집힌 죽은 단언. `toHaveStyle`은 jest-dom이 expected hex를 같은 CSSOM으로 `rgb(...)` 정규화해 like-for-like 비교(probe로 양쪽 정규화값 동일 확인). 텍스트 없는 장식 `<span aria-hidden="true">`(예: 비교 매트릭스 헤더 색 스와치)는 role/text 쿼리 불가 → `container.querySelector('span[aria-hidden="true"]')`(또는 `thead span[aria-hidden]`)로 찾아 `toHaveStyle`. `data-testid`/`title` 추가 지양(title=사용자 노출→ko.ts 필요). 동치 대안: 참조 span에 `ref.style.backgroundColor=hex` 후 `swatch.style.backgroundColor===ref.style.backgroundColor`.
- **React key 충돌 회귀 테스트는 row 개수 단언이 아니라 `console.error` 스파이로** (step-level SLO Task 7): React는 중복 `key`여도 모든 row를 렌더하고 dev `console.error`(`...same key...`)만 낸다 — `getAllByText(/x/).length >= N`은 버그(중복 key) 코드에서도 통과해 fix를 증명 못 한다. `vi.spyOn(console,"error")` 후 `mock.calls`에 `"same key"` 포함 여부를 단언(`toHaveBeenCalledWith` 매처보다 robust — format-string/interpolated-arg 차이에 안 깨짐). **teeth-check 필수**: 키를 일부러 버그값으로 되돌려 FAIL 확인 후 복원.
- **blob URL 누수** (Slice 5): `URL.createObjectURL` 결과는 명시적 `revokeObjectURL` 전까지 페이지 lifetime 내내 남는다. 컴포넌트면 `useEffect cleanup`, fire-and-forget 헬퍼면 `setTimeout(()=>revokeObjectURL,1000)`로 revoke. (2026-06 result-screen-polish: `DownloadJsonButton` 제거→`api/downloadJson.ts` 헬퍼로 추출·setTimeout-지연 revoke·`downloadJson.test.ts`가 fake timers로 1회 revoke 검증.)
- **jsdom은 `URL.createObjectURL`을 구현하지 않음** (Slice 5): `downloadJson`/`ReportView` 테스트에서 `Object.defineProperty(URL, "createObjectURL", ...)` 폴리필을 모듈 스코프에 추가. 폴리필은 conditional (`typeof URL.createObjectURL === "undefined"`)로.
- **Chrome `<a href="blob:..." download>` 가 가끔 "Check Internet Connection"으로 실패** (Slice 5): **온라인에서도** 가끔 발생, 보통 retry로 해소되는 transient 실패(원인 추정: Chrome 다운로드 매니저의 Safe Browsing phone-home). 우회: **`window.showSaveFilePicker`**(File System Access API, Chrome/Edge 86+)는 다운로드 매니저를 거치지 않아 무관하고 저장 위치를 고를 수 있어 UX도 낫다. Firefox/Safari 미지원이라 blob URL anchor click fallback 유지. 패턴: `if (typeof window.showSaveFilePicker === "function") { ... } else { /* blob URL anchor click */ }`. (처음 "에어갭"으로 오진했었음 — 같은 함정 또 만나면 transient/retry 먼저 의심.)
- **네이티브 Window 메서드(`showSaveFilePicker`)는 *바운드* 호출 — 변수로 빼서 부르면 Chrome `Illegal invocation`** (result-screen-polish): `const picker = window.showSaveFilePicker; await picker(opts)`는 `this`가 분리돼 실제 Chrome에서 `TypeError: Illegal invocation`(detached `fetch`/`alert`과 동류)으로 throw → try/catch가 삼키면 매번 blob 폴백으로 빠져 picker 경로가 死(에어갭 시나리오 무력). 반드시 `picker.call(window, opts)`(또는 `window.showSaveFilePicker(opts)` 직접). **`vi.fn()` 목은 `this`를 안 보므로 RTL이 이 버그를 못 잡는다** — 코드리뷰/실브라우저로만 발견(이번엔 T2 코드리뷰가 적발). `api/downloadJson.ts`가 `picker.call(window, …)`로 구현.
