넓게 보기 — 스텝을 어떤 형태로 펼칠까요?

토글을 켜면 디테일 열이 사라지고 스텝 영역이 전체 폭을 씁니다(스텝 클릭 → 모달 편집, 합의됨). 남은 질문: 넓어진 공간에 스텝을 어떻게 배치할지입니다. 실제 시나리오(로그인 → 반복 → 조건 → 로그아웃)로 두 안을 그렸습니다.

A

전폭 세로 리스트 — 지금 순서 그대로, 한 줄에 정보를 다 편다

지금의 세로 아웃라인이 전체 폭으로 늘어나고, 좁아서 잘리던 URL 전체 + 부가 정보 칩(검증·추출·think time)이 한 줄에 보입니다. 실행 순서 = 위→아래 그대로.

넓게 보기 ON — 세로 리스트가 전체 폭 사용
⠿POST로그인https://shop.example.com/api/auth/login검증 1추출 token
⠿상품 브라우징반복 ×10
⠿GET상품 목록https://shop.example.com/api/products?page={{page}}&sort=recent&size=50검증 2think 200–500ms
⠿GET상품 상세https://shop.example.com/api/products/{{product_id}}추출 price
⠿POST로그아웃https://shop.example.com/api/auth/logout

장점

  • 실행 순서 읽기가 그대로 (위→아래)
  • 드래그 재정렬이 와이드 모드에서도 그대로 동작 (현 드래그 구현이 세로 리스트 전제)
  • URL·검증·추출을 클릭 없이 훑을 수 있음 — 시나리오 검토에 강함

단점

  • 스텝이 많으면 세로 스크롤은 여전 (한 화면에 보이는 스텝 수는 동일)
B

가로 카드 플로우 — 예전 캔버스처럼 왼→오른쪽으로 흐른다

구 캔버스(가로 배치) 감각. 최상위 스텝이 카드로 왼→오른쪽으로 흐르고 화면 끝에서 다음 줄로 랩. 반복/조건 컨테이너는 자식을 카드 안에 세로로 품습니다. 한 화면에 더 많은 스텝이 들어옵니다.

넓게 보기 ON — 가로 카드 플로우
POST로그인…/api/auth/login
→
상품 브라우징 반복 ×10
GET상품 목록 …/products?page=…
GET상품 상세 …/products/{{product_id}}
→
POST로그아웃…/api/auth/logout

장점

  • 한 화면에 들어오는 스텝 수가 가장 많음 — 긴 시나리오 전체 조망
  • 구 캔버스의 "흐름을 본다"는 감각 복원

단점

  • 줄바꿈(랩) 지점에서 읽기 순서가 꺾여 Z자로 읽어야 함
  • 카드 폭이 좁아 URL은 다시 잘림 (칸이 좁아서 A만큼 정보를 못 실음)
  • 드래그 재정렬 재작업 필요 — 현 드래그(세로 전제)를 와이드에서 끄고 시작해야 함
  • 세로 아웃라인과 배치가 완전히 달라져 모드 전환 시 위치 감각 리셋

클릭해서 선택해 주세요. 제 추천은 A입니다 — "넓게 보기"의 목적이 스텝 내용을 더 보는 것이라면 A가 정확히 그걸 주고, 드래그·읽기 순서도 그대로라 구현 위험이 낮습니다. B는 "전체 흐름 조망"이 목적일 때 좋지만 URL이 다시 잘려서 이번 불편(위아래 왔다갔다)의 해결과는 거리가 있습니다. 다른 아이디어가 있으면 터미널로 알려주세요.
