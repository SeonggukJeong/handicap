# 0009. MVP에 라이브 대시보드 없음

- **상태**: Accepted
- **날짜**: 2026-05-27

## Context

부하 테스트 도구는 보통 테스트 진행 중 실시간 그래프(RPS, 응답시간, 에러율 시계열)를 보여준다. 이 라이브 스트리밍은 WebSocket/SSE + 시계열 집계 + 실시간 차트 인프라를 모두 요구한다. MVP에 포함할지가 쟁점.

## Decision Drivers

- 사내에 이미 운영 중인 관측 도구가 있는가
- LoadRunner 대비 차별점이 어디에 있는가 (사용자 인터뷰 결과)
- MVP의 구현 비용 절감 가능성
- 안전 장치 필요성 (테스트 중 이상 시 자동 중단 등)

## Considered Options

1. **풀 라이브 대시보드** — 모든 지표 실시간 차트, 자동 이상 감지·중단
2. **주요 지표만 라이브** — 진행률·현재 RPS·에러 카운트 수치만, 차트는 종료 후
3. **종료 후 리포트만** — 라이브 스트리밍 인프라 전면 제외

## Decision

**옵션 3: 종료 후 리포트만.** APM(Datadog/Grafana)이 이미 운영 중이라 이상 감지·실시간 관측은 그쪽에 위임. 사용자가 LoadRunner 대비 좋다고 평가한 것은 "상세 리포트" 였지 "실시간 차트" 가 아니다. 라이브 인프라 통째로 빠지면 MVP 범위가 크게 줄어든다.

## Consequences

**Positive**
- WebSocket/SSE, 시계열 DB(InfluxDB/Prometheus), 라이브 차트 라이브러리 등 인프라 전부 MVP에서 제외
- 리포트 깊이(LoadRunner급)에 자원 집중
- 테스트 종료 후 정확한 집계 리포트가 차별점

**Negative / Trade-offs**
- 테스트 진행 중에는 "진행률 %" 외에 거의 보여줄 게 없음 — 긴 테스트(30분+) 에서 답답할 수 있음
- 안전 장치(에러율 급증 시 자동 중단)는 별도로 다뤄야 함 → 진행 중 컨트롤러가 워커 raw counter를 폴링하여 임계값 초과 시 중단하는 minimal 안전장치는 포함 권장
- "실시간 보고 싶다" 요청이 들어오면 후속 단계에서 추가 (옵션 2 정도로)
